4 Quiz Website
4.1 Objective:
Create an online quiz platform that allows users to take quizzes, view their scores, and track their progress
over time through a dynamic web interface. The application will support features such as timed quizzes,
random question selection, and user accounts to enhance the learning experience and provide feedback.

4.2 Functional Requirements:
1. The website should allow users to create accounts, log in, and log out securely, enabling them to save
their quiz progress and scores.
2. The website should allow admin users to create, edit, and delete quizzes, including the ability to add
questions with various formats (multiple choice, true/false, etc.) and set time limits for completion.
3. The website should allow users to take quizzes, with a user-friendly interface that displays questions,
tracks time, and allows for navigation between questions before submission.
4. The website should automatically calculate and display usersâ€™ scores upon quiz completion, providing
immediate feedback on correct and incorrect answers along with explanations where applicable.
5. The website should allow users to have access to a personal dashboard that shows their completed
quizzes, scores, and overall progress over time, allowing them to track improvements and revisit previous
quizzes.

4.3 Non-Functional Requirements:
1. The website should be developed with HTML, CSS, and JavaScript (frontend). Server-side languages,
such as PHP, are also allowed.
2. The website should be developed using a popular backend framework (Node.js, Django, Flask, Ruby
on Rails, etc.).
3. The website should be hosted on a popular web server framework (Apache HTTP Server, Nginx,
Microsoft Internet Information Services, Tomcat, Node.js, H2O, etc.)
4. The website should load within two seconds on standard high-speed internet connections.
5. The website should be optimized for fast response times, ensuring a smooth user experience.
6. The website should ensure secure all data transmissions.
7. All user inputs should be validated and sanitized to prevent security vulnerabilities.
8. The website should function consistently across major web browsers (Chrome, Firefox, Safari, Edge).
